Data Filters¶

In many scenarios, it is required for a Business Unit to be limited to a specific list of parameter values. This can prevent a Business Unit to access data that belongs to a different Business Unit.

Business Unit filters can be managed only by system administrators.

  • Calculation Parameter Type

    Required
    Type of parameter to filter. Only enumerable parameter types are available here.

  • Value

    Explicitly stating list of allowed values.

    To separate each individual value Item Separator, configured in associated calculation parameter type, has to be used. See Calculation Parameter Types for more info.

  • Wildcard Value

    When a Wildcard is provided, the system automatically passes all values that match the wildcard.

    Table 1 Special Wildcard Characters¶

    Character

    Description

    Example

    %

    Represents zero or more characters

    bl% finds bl, black, blue, and blob

    _

    Represents a single character

    h_t finds hot, hat, and hit

    []

    Represents any single character within the brackets

    h[oa]t finds hot and hat, but not hit

    ^

    Represents any character not in the brackets

    h[^oa]t finds hit, but not hot and hat

    -

    Represents a range of characters

    c[a-b]t finds cat and cbt

    For example: When you’re searching for every support team, then write %support% into Wildcard value

    To separate each individual wildcard value Item Separator, configured in associated calculation parameter type, has to be used. See Calculation Parameter Types chapter for more information.

Specifying a filter without entering a value nor wildcard value means that no values will be available to the Business Unit.
