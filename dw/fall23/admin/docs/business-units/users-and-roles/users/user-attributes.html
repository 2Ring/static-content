User Attributes¶

User attributes are used to filter displayed grid based values differently for different logged in users by using them in dynamic filter. Filter is represented by expression with placeholders which are replaced with value from user attributes. For more information, see Filtering using User Attributes chapter.

To initialize user attributes, it is necessary to create following layout:

  • Layout must be in the System BU and must be named “UserAttributes”

  • Single grid shared calculation has to be configured in this layout

    • Grid calculation function, whose data should be available as user attributes, has to be selected as a source table. Therefore, this grid calculation function should contain user related data.

  • A custom column, with the “Text” DataType and “Username” or/and “UserAttributeKey” name has to be configured on this calculation

    • “Username” is paired to the DW user name of the user

    • “UserAttributeKey” is paired to the UserAttributeKey value on the user (configurable on the user entity)

    • If both, the “Username” and “UserAttributeKey” custom columns are configured:

      1. user that matches both of them is paired.

      2. if there is no user who matches both columns, user who matches the “UserAttributeKey” is paired

      3. if there is no user who matches the “UserAttributeKey”, user who matches the “Username” is paired

  • This grid shared calculation must be used in the grid and all columns that should be used as user attributes (also “Username” and “UserAttributeKey”) should be configured

User attributes are available for the selection in the dynamic filter after publishing this layout.
