Grid Calculation FunctionsÂ¶

A Grid calculation function is a named method for calculating Grid values and is associated with a stored procedure in the connector database.

Adding new Grid calculation functions to an existing 2Ring connectors is not possible but custom derived Grid columns can be created.

  • Name

    Required
    The name of the grid calculation function.

  • Display Name

    A name that appears as the segment title when the grid calculation function is selected as a source table. If the display name is not filled, the title is automatically set from the grid calculation function name in which all words are separated by a space and all non-alphanumeric characters are also replaced by a space.

  • Description

    A description of the Grid Calculation Functions which appears in dropdown tooltips in both Administration Application and Layout Editor.

  • Merge Columns

    Columns that uniquely identifies a record.

  • Default Columns

    Columns that are automatically added to the grid when the grid calculation function is selected in the layout editor.

  • Default Client Sorting Columns

    Columns that are automatically set as a sorting columns in ascending order when the grid calculation function is selected in the layout editor.

  • Total Row Label

    The label of the totals row.

  • Show Total Label in Column

    Column, in which the total label is displayed. Must be included in Default Columns.

  • Metadata

    A user can view or edit a metadata.

  • Capabilities

    Capabilities of the connector, defined during the installation.

    • Lock Built-In Columns

      Read-only
      If enabled, administrator can not add, delete or edit any built-in columns.

    • Lock Parameters

      Read-only
      If enabled, administrator can not add, delete or edit any grid calculation parameters.

    • Include Used Columns In Formula Hash

      Read-only
      Whether two grid-based sources differing only in the configured columns should each have its own calculation generated.

    • Support Intervals

      Read-only
      Whether connector supports interval based grid calculations. Similarly to interval based KPIs, this allows historical values to be displayed in grids.

    • Support Timezone

      Read-only
      Whether connector supports timezone configuration in grid calculations.
