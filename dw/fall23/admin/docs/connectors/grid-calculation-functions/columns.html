Columns¶

Custom computed (derived) Grid columns can be added to Grid calculation functions. Derived columns offer the flexibility to add columns that do not come with the connector when it is installed.

Derived Grid columns cannot be used in other derived columns.

Custom functions dbo.MinNumeric(), dbo.MaxNumeric(), dbo.MinDateTime() and dbo.MaxDateTime() can be used to compare two values to return either the lesser or greater one.

  • Name

    Required
    The name of the Grid calculation function column.

  • Display Name

    A name that appears as the grid column’s header when the grid calculation function column is selected as column type. If the display name is not filled, the header is automatically set from the grid calculation function column’s name in which all words are separated by a space and all non-alphanumeric characters are also replaced by a space.

  • Description

    A description of the Grid calculation function Column which appears in dropdown tooltips in both Administration Application and Layout Editor.

  • Data Type

    Required
    Indicates the type of value associated with this column.|br| Available options are Numeric, Text, DateTime and TimeSpan.

  • Storage Data Type

    Actual column type in SQL database.
    List of all supported types is available here.

  • Calculation Parameter Type

    Determines a calculation parameter type whose values appear in this column.

  • Value Type

    Whether calcualtion parameter type name or calculation parameter type value appears in this column.

  • Aggregation Function

    A method how to merge values if multiple rows are returned. This property also defines how the total value of the column is calculated.
    Available options are Average, Count, Count Distinct, Count Not Null, First, Last, Max, Median, Min, Standard Deviation, Sum, Variance and Weighted Average

  • Is Derived

    Required
    Whether this column is natively calculated by connector or if its additionally calculated (derived) from existing columns. For derived columns Expression has to be provided.

  • Expression

    Available only when column is derived.
    T-SQL expression to compute the column. Other column values can be used in the expression by using format column_name. Simple or advanced SQL logic can be used in the expression. This includes simple math operations, cast operations, etc. If a number with a decimal point is desired, the expressions have to be multiplied by 1.0.

    Example: Create a message column that displays “Please end the call” when an agent is talking over 15 minutes. The expression would be:

    CASE
    WHEN Duration > 900 AND State = 'Talking'
    THEN 'Please end the call.'
END

    If the “DateTime” is used as a data type for a derived column, expression has to evaluate to the Unix Time format in milliseconds. For more information about the Unix Time and conversion to Unix Time, visit this website.

    If the created expression is invalid, all Grids using that Grid calculation function will become invalid as well.

  • Default Settings

    A JSON field that contains the default settings for entities using the Grid calculation function column. These settings are applied when the Grid calculation function column is selected. The first level contains a type of the entity and the second level contains default settings for the associated entity type.
    All available settings are listed in the Default Settings - Schema chapter.

    Example :

    {
    "gridColumn": {
        "widthRatio": 50,
        "displayFormat": "{0:F0}",
        "horizontalAlignment": "Right",
        "presentationType": "Alphanumeric"
    },
    "barLineChartSeries": {
        "displayFormat": "{0:F0}"
    }
}
  • Metadata

    A user can view or edit a metadata.
