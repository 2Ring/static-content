Notification Channel Types¶

Each Notification Channel has Type associated with it. Notification Channel Type defines available settings for Channel and also contains setup of request to external system represented by channel.

To setup notifications, a notification channel has to exist in the Business Unit and it’s type has to be set to one of the available notification channel types. Notifications are associated with the notification channel, not a notification channel type. For more information, see Notification Channels chapter.

  • Name

    Required
    The name of a notification channel type.

  • Verb

    Required
    Action of the request.

    Possible values are Post, Get, Put and Patch.

  • Body Format

    Required
    The format of the data being sent.

    Possible values are JSON, Text, XML and URL Encoded.

  • URL

    Required
    Web service URL.a

  • Ignore Certificate Errors

    Required
    Calling a web service hosted on https may cause errors if the certificate is not trusted on the server where DW is hosted. To ignore these errors, turn on ignore certificate errors.

  • Recipient Address Pattern

    Some notification channels have a predefined format for message recipients. For example email address have to contain @ character and domain. These requirements can be set here and will be validated when recipients will be created for this channel.

  • Header

    Http headers of the web service request. DW placeholders can be put into the header of message. List of all placeholders is available here.

  • Max Body Length

    Maximum number of characters message body can contain.

  • Body

    Http body of the web service request. DW placeholders can be put into the body of message. List of all placeholders is available here.
