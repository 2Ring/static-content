2.1. Queues Statistics Current¶

Depending on the KPI function, the following parameters may be used:

  • Queues – list of queue names.

2.1.1. Queues Statistics Current - Agents After Contact Work¶

This KPI function returns the number of agents who are doing After Call Work for a contact.

Parameters: Queues

2.1.2. Queues Statistics Current - Agents Available¶

This KPI function returns the number of agents who have set their status to Available.

Parameters: Queues

2.1.3. Queues Statistics Current - Agents Error¶

This KPI function returns the number of agents who have their contact in an error state. Parameters: Queues

2.1.4. Queues Statistics Current - Agents Non Productive¶

This KPI function returns the number of agents who have set their status to a custom status (other than Available or Offline).

Parameters: Queues

Agents can handle contacts while their status is set to a custom status. For example, agents can be On call or doing ACW while their status is set to a custom. This means it’s possible for agents to be counted as On call and Non Productive at the same time.

2.1.5. Queues Statistics Current - Agents On Call¶

This KPI function returns the number of agents currently on a contact.

Parameters: Queues

2.1.6. Queues Statistics Current - Agents Online¶

This KPI function returns the number of agents who have set their status to something other than Offline.

Parameters: Queues

2.1.7. Queues Statistics Current - Agents Staffed¶

This KPI function returns the number of agents who are online, and not in Non Productive (a custom status).

Parameters: Queues

2.1.8. Queues Statistics Current - Contacts In Queue¶

This KPI function returns the number of contacts currently in the queue.

Parameters: Queues

2.1.9. Queues Statistics Current - Contacts Scheduled¶

This KPI function returns the number of customers in the queue for which there is a callback scheduled.

Parameters: Queues

2.1.10. Queues Statistics Current - Oldest Contact Age¶

This KPI function returns the time of the contact that has been in the queue the longest.

Parameters: Queues
