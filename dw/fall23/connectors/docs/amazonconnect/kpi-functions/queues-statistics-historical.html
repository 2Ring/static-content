2.2. Queues Statistics Historical¶

Depending on the KPI function, the following parameters may be used:

  • Queues – list of queue names.

  • Threshold – time in seconds. Calls with queue time lower than the threshold will be considered as handled in service level. The possible threshold values are: whole numbers between 1 and 604800(inclusive).

You have to select exactly one threshold value, otherwise KPIs using the parameter will display an error.

2.2.1. Queues Statistics Historical - Api Contacts Handled¶

This KPI function returns the number of contacts that were initiated using an Amazon Connect API operation and handled by an agent.

Parameters: Queues

2.2.2. Queues Statistics Historical - Average Abandon Time¶

This KPI function returns the average time that contacts waited in the queue before being abandoned.

Parameters: Queues

2.2.3. Queues Statistics Historical - Average After Contact Work Time¶

This KPI function returns the average time that an agent spent in the After Contact Work (ACW) status after handling a contact. Agents enter the ACW status after a contact ends and leave it when they select a different agent status.

Parameters: Queues

2.2.4. Queues Statistics Historical - Average Handle Time¶

This KPI function returns the average time that an agent spent on contacts, including hold time and after contact work time.

Parameters: Queues

2.2.5. Queues Statistics Historical - Average Hold Time¶

This KPI function returns the average time that customers spent on hold after being connected to an agent.

Parameters: Queues

2.2.6. Queues Statistics Historical - Average Interaction And Hold Time¶

This KPI function returns the average of the sum of the interaction time and hold time.

Parameters: Queues

2.2.7. Queues Statistics Historical - Average Interaction Time¶

This KPI function returns the average time that agents spent interacting with customers on a contact.

Parameters: Queues

2.2.8. Queues Statistics Historical - Average Occupancy¶

This KPI function returns the percentage of time that agents were active on contacts.

Parameters: Queues

2.2.9. Queues Statistics Historical - Average Queue Answer Time¶

This KPI function returns the average time that contacts waited in the queue before being answered by an agent.

Parameters: Queues

2.2.10. Queues Statistics Historical - Callback Contacts Handled¶

This KPI function returns the number of contacts that were initiated from a queued callback and handled by an agent.

Parameters: Queues

2.2.11. Queues Statistics Historical - Contacts Abandoned¶

This KPI function returns the number of contacts disconnected by the customer while in the queue.

Parameters: Queues

Contacts queued for callback are not counted as abandoned.

2.2.12. Queues Statistics Historical - Contacts Agent Hung Up First¶

This Kpi function returns the number of contacts disconnected where the agent disconnected before the customer.

Parameters: Queues

2.2.13. Queues Statistics Historical - Contacts Consulted¶

This KPI function returns the number of contacts disconnected where the agent disconnected before the customer.

Parameters: Queues

2.2.14. Queues Statistics Historical - Contacts Handled¶

This KPI function returns the number of contacts that were connected to an agent.

Parameters: Queues

2.2.15. Queues Statistics Historical - Contacts Handled Incoming¶

This KPI function returns the number of incoming contacts that were handled by an agent, including inbound contacts, transferred contacts, and scheduled callbacks.

Parameters: Queues

2.2.16. Queues Statistics Historical - Contacts Handled Outbound¶

This KPI function returns the number of outbound contacts that were handled by an agent.

Parameters: Queues

2.2.17. Queues Statistics Historical - Contacts Hold Abandons¶

This KPI function returns the number of contacts that disconnected while the customer was on hold.

Parameters: Queues

2.2.18. Queues Statistics Historical - Contacts Missed¶

This KPI function returns the number of contacts routed to an agent but not answered by the agent, including contacts abandoned by the customer, during the specified time range.

Parameters: Queues

A contact can be counted as missed multiple times, once for each time it is routed to an agent but not answered.

2.2.19. Queues Statistics Historical - Contacts Queued¶

This KPI function returns the number of contacts placed in the queue.

Parameters: Queues

2.2.20. Queues Statistics Historical - Contacts Transferred In¶

This KPI function returns the number of contacts transferred to the queue by an agent.

Parameters: Queues

2.2.21. Queues Statistics Historical - Contacts Transferred In From Queue¶

This KPI function returns the number of contacts transferred to the queue by an agent.

Parameters: Queues

2.2.22. Queues Statistics Historical - Contacts Transferred Out¶

This KPI function returns the number of contacts transferred from the queue after being answered by an agent.

Parameters: Queues

2.2.23. Queues Statistics Historical - Contacts Transferred Out From Queue¶

This KPI function returns the number of contacts transferred from the queue to another queue.

Parameters: Queues

2.2.24. Queues Statistics Historical - Longest Queued Time¶

This KPI function returns the longest time that a contact spent waiting in the queue.

Parameters: Queues

2.2.25. Queues Statistics Historical - Service Level¶

This KPI function returns the percentage of contacts removed from the queue between 0 and X seconds after being added to it.

Parameters: Queues, Threshold

Maximum of 20 unique SERVICE_LEVEL metrics can be included in a request.
