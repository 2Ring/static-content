Connector Installation¶

Before the installation, make sure the connector version is compatible with the DW version. DW and its connectors are considered to be compatible, when both minor and major version numbers match.

Example:
- DW 8.0.1 is compatible with connector 8.0.3
- DW 8.1.0 is NOT compatible with connector 8.0.3
- DW 8.0.1 is NOT compatible with connector 8.1.0

Extract 2Ring_DW_DEMO_Connector_vX.Y.Z.zip file – vX.Y.Z indicates the version of the connector. It is recommended to place the files in C:\Program Files\2Ring\DW\Connectors\XYZ.

Do not remove the unzipped package after the installation.

To perform an action such as install or uninstall you need to supply action specific parameters either in an xml file or later at runtime once prompted. Most frequent actions have a pre-created parameters.xml file prepared in the distribution package.

  • Required parameters that are not in an XML file or their value is left blank will be prompted for value during runtime.

  • Optional parameters that are not in an XML file will be prompted for value during runtime. If an optional parameter value is left blank, the parameter will be ignored.

  • Passwords entered in the xml files will not get displayed during runtime.

  • During runtime, some parameters may require confirmation. The user will be asked to enter the value twice.

To prepare for install, uninstall or upgrade, follow these steps:

  1. Configure connection to SQL Server instance where DW database is installed.

    • Open Connection.xml

    <ServerName></ServerName>
<DBName></DBName>
<UserName></UserName>
<Password></Password>
  2. Edit the following fields if necessary:
    • ServerName – the address (hostname or IP address\InstanceName) of the SQL server, that contains the DW database

      If the SQL Server does not run on the default port, use format: hostname or IP address\InstanceName, Port

    • DBName – the name for the Demo Connector database

    • UserName (Optional) – the username of SQL user that will be used to create the Demo connector database

      Leave blank to use integrated security mode. The user has to have the sysadmin role assigned in the SQL Server.

    • Password (Optional) – the password of the SQL user

      If left blank, the user will be prompted to enter the password at runtime. If the UserName is left blank, the Password must be blank as well.,

  3. Save the file.

  4. Open InstallParameters.xml
    <Parameter Name="ConnectorName" Value="Demo"/>
<Parameter Name="DWDatabase" Value="DW"/>
  5. Edit the following fields if necessary:

    • ConnectorName – the name of the connector

    • DWDatabase – the name of the DW database

    Example of filled in parameters:

    <Parameter Name="ConnectorName" Value="Demo"/>
<Parameter Name="DWDatabase" Value="DW"/>

  6. Save the file

  7. Open the command prompt as Administrator and navigate to the extracted directory

  8. Run Install.cmd

  9. Run InstallService.cmd

    • Enter the name and password of the DW Application user once prompted

    • Enter the display name for the connector service once prompted

Documentation Update

In case of installing, or upgrading a connector, which is a higher version than the currently installed version of 2Ring Dashboards and Wallboards, a manual documentation update is recommended.
To do that, open the documentation zip-file, which is provided with the connector, and extract the files in the DW deploy folder -> docs -> connectors. Replace the files in the destination, if prompted.
