3. SQL PatternsÂ¶

It is possible to use Transact-SQL patterns in the Wildcards section of some parameters. For more information about SQL patterns see https://msdn.microsoft.com/en-us/library/ms179859.aspx.

Filters, where you can use SQL Patterns:

  • Agents

  • AgentStates

  • Skills - Allows you to select from a list of skills using a logical OR operation for filtering. When you use this parameter, any object that matches at least one of the selected skills will be included in the results.

  • Teams

It is possible to use more than one SQL Pattern by creating a comma separated list. E.g.: CSQ_A%, CSQ_B%, CSQ_C%
