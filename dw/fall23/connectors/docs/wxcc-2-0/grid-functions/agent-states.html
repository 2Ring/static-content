2.1. Agent States¶

The Agent States Grid Function displays the current Agent activity in a tabular format.

For more information, see the Webex API documentation.

This Grid Function utilizes REST API for initialization and Webhook Notifications for updates.

2.1.1. Columns¶

  • Agent Id - the Id of the agent

  • Current State - the current state the agent is in

  • Duration - the amount of time the agent spent in the current state

  • Duration Since Last State Change - the amount of time since the agent’s last state change

  • Reason - the reason code of the current state

  • Start Time - timestamp of when the agent entered the current state

  • Start Time Since Last State Change - timestamp of the agent’s last state change

  • Task Id - the Id of the task the agent is currently handling

2.1.2. Calculation Parameters¶

The Grid function has these optional parameters:

  • Agents - list of Agent names

  • CompetencyLevel - Minimum profiency level of skill in skill profile, for skill type PROFIENCY

  • Queues - List of Queue names

  • ReasonCodes - List of Reason Codes

  • Skills - List of Skill names

  • SkillsBooleanValue - Boolean value of skill in skill profile, for skill type BOOLEAN

  • SkillsEnumValues - List of values of skill in skill profile, for skill type ENUM

  • SkillsTextValues - List of values of skill in skill profile, for skill type TEXT

If the Skill parameter is filled, it is necessary to fill in some of the parameters SkillsBooleanValue, SkillsEnumValues, SkillsTextValues and CompetencyLevel according to the type of skills specified in the parameter.
The agent will be displayed if assigned a skill profile that includes any of the skills specified in the parameter, combined with the skill value according to the skill type.

2.1.3. Column-mapped Parameters¶

  • Agent Id - Agents calculation parameter value

  • Current State - State calculation parameter value
