2.5. Queue Agent States¶

The Queue Agent States displays the last Agent activity for the current day grouped by Queues in a tabular format.

For more information, see the Webex API documentation.

This Grid Function utilizes REST API for initialization and Webhook Notifications for updates.

2.5.1. Columns¶

  • Channel Type - the type of channel associated with the queue

  • Queue Id - the id of the queue

  • Queue Name - the name of the queue

  • Available - the number of agents in the available state and associated with the queue

  • Conferencing - the number of agents in the conferencing state handling a task associated with the queue

  • Connected - the number of agents in the connected state handling a task associated with the queue

  • Consulting - the number of agents in the consulting state handling a task associated with the queue

  • Hold - the number of agents in the hold state handling a task associated with the queue

  • Idle - the number of agents in the idle state and associated with the queue

  • Not Responding - the number of agents in the not responding state handling a task associated with the queue

  • Offered - the number of agents in the offered state handling a task associated with the queue

  • Wrapup - the number of agents in the wrapup state handling a task associated with the queue

2.5.2. Calculation Parameters¶

The Grid function has these optional parameters:

  • Channel Type - list of Channel Types

  • Queues - list of Queue names

  • Queues Groups - list of parameter groups. This parameter represents groups of parameters, which will be used in the calculation of this function

2.5.3. Column-mapped Parameters¶

  • Channel Type - Channel Type calculation parameter value

  • Queue Id - Queues calculation parameter value

  • Queue Name - Queues calculation parameter value
