1.4. Queue Agent States¶

The Queue Agent States KPI Functions display aggregated last Agent activities for the current day grouped by Queues.

For more information, see the Webex API documentation.

These KPI Functions utilize REST API for initialization and Webhook Notifications for updates.

Depending on the KPI Function, the following parameters may be used:

  • Queues - list of Queue names

  • Channel Type - list of Channel Types

  • State - list of States

1.4.1. Queue Agent States - In State¶

This KPI Function returns the number of agents that are currently in the specified states. Task state, such as Hold or Conferencing, is not taken into account.

Parameters: Queues, Channel Type, State
